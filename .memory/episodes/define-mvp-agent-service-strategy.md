# Define MVP Agent Service Strategy

The user clarified that Chronelle's main purpose is to speed up development of
other projects, not merely to exist as a memory ontology or CLI utility.

The discussion explored whether Chronelle should remain a CLI/tool, become a
local agent service, or be built directly as a hosted vendable agent service. The
working framing is that Chronelle should pursue the agent-service business
direction while keeping project-owned `.memory` files and a simple core API/CLI
as the trust layer underneath.
