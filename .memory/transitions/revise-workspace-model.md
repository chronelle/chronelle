# Revise Workspace Model

## Rationale

The workspace model needed to show the `.memory/` convention and Chronelle's
narrowed scope.

## Before

The workspace model listed Ontology, Projects, and Projections under one
workspace, without saying where project memory lives.

## After

The workspace model shows Chronelle holding the shared ontology, projections, and
tooling alongside its own `.memory/`, and shows each project keeping memory in its
own `.memory/` directory.
