# Create Episode and Transition Decision

Chronelle recorded an accepted project Decision to model meaningful activity as
Episodes and state change as Transitions.

## Rationale

Because Chronelle is being built with Chronelle itself, the ontology update
should also appear in project memory.

## Before

The Chronelle project did not record a Decision for its temporal primitives.

## After

The Chronelle project records `episode-transition-primitives` as an accepted
Decision.
